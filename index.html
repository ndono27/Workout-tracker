Workout Tracker

This project is a browser-based workout management app that lets users create custom training templates, run active workout sessions, log sets with reps and weight, track completed workouts by date, and monitor personal bests over time.

Account System

  • Create account and login flow per user
  • User-specific workout data separation
  • Persistent active user session
  • Account menu actions: logout

Workout Builder

  • Create named workout templates
  • Add standard exercises with planned sets
  • Build supersets with multiple linked exercises
  • Edit/delete templates and start them for selected dates

Performance Tracking

  • Active workout logging for reps and optional weight
  • Workout completion summary with duration
  • Automated personal best detection and updates
  • Grouped completed-workout history with full set details

Core App Architecture

The app uses a central in-memory state object for current user, workouts, calendar entries, completed-day markers, personal best records, and active sessions. UI is updated by direct DOM rendering functions, with specialized renderers for each tab and feature area.

  • State-driven rendering approach
  • Modular helper functions for formatting and normalization
  • Input validation for required workout and set fields
  • Event-based tab switching and action handling

Calendar + Session Engine

Workouts are scheduled and completed by day key (YYYY-MM-DD). Starting a workout creates an active session from a saved template. Completing a session writes a normalized entry into that day, marks the day complete, and recalculates metrics where needed.

  • Date-keyed workout assignment and history
  • Session-based set log capture
  • Completion marker visualization in calendar
  • Template-linked cleanup when workouts are deleted

User Journey

  1. Create account or log in
  2. Build one or more workout templates (including supersets)
  3. Select a day and start a saved workout
  4. Enter reps/weight for each set in Active Workout
  5. Complete workout and review summary/personal-best updates
  6. Inspect history and performance trends in Completed and PB tabs

Data Model + Persistence

Data is persisted in LocalStorage under user and session keys. Each user record stores credentials and a nested data object for workouts, calendar entries, completed-day flags, and personal bests.

  • Multi-user records in one browser profile
  • Active user key for auto-login continuity
  • Personal best aggregation by workout and exercise
  • Safe fallback defaults when malformed data is detected
